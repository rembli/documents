About

This is a demo web site for a simple web application.

The purpose of this web application is to demonstrate some techniques of 2015 web development. The application is written in Java, SQL, JavaScript, HTML and CSS using the following frameworks, libraries and tools: Bootstrap, jQuery, Dust.js, Swagger, JAX-RS/Jersey, JAX-B, Tomcat, Jackson, Genson, Lombok, Mimepull, SQL2O, Hikary, mySQL.

For some special document related tasks I used Dropzone (Drag 'n Drop file upload) and Thumbnailator (nice thumbnails).

'Features' that this web application for demonstration purposes includes are:

  • Rich UI client built with HTML/CSS/Javascript using Bootstrap as grid-system and Dust as template engine (incl. deep-linking)
  • Tags for client-side-includes of html-templates (e.g. footer) and tags to translate labels (and even content)
  • ROCA-style system design with smooth transitions of a SPA (Single-page-application)
  • REST-API layer built with JAX-RS and API documentation using Swagger (incl. Try-it-out and dedicated resources different from backend-model)
  • Handling of multi-part file upload using Drag and Drop and a demo of a web scanner
  • Token-based authentication for back-end and front-end and a "Login with facebook"-button
  • Clear separation between back-end in Java and SQL, API-layer with REST and front-end with HTML and JS
  • Simple and effective database persistence using SQL2O and a 'domain'-oriented system design
Focus of the application design was:

  • Code should be concise, easy to read and to maintain ('read the code like a book')
  • Leave the developer under control; i.e. not using too much 'magic'-frameworks
  • Design each layer 'API-first': What would be the other guys requirements for an easy to use interface?
  • Give the specific developer roles what they want: HTML/CSS for front-end; Java for back-end; not one-size-fits-all
  • Design-oriented: From screen to API to features and functions
Take a look and try the API with Swagger
